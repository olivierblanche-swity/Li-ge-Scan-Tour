Conversation 1 - Contexte projet initial
----------------------------------------

Utilisateur :

Le projet Liège Scan Tour est un guide touristique consacré aux monuments de la ville de Liège.

Fonctionnalité principale :
1. l'utilisateur arrive sur la page d'accueil ;
2. clique sur "Prendre une photo" ;
3. accède à la caméra de son appareil ;
4. prend une photo ;
5. le système analyse la photo ;
6. le monument est identifié ou confirmé ;
7. les informations du monument sont affichées.

Le scanner est la fonctionnalité centrale du projet.
Toutes les décisions UX doivent mettre en avant le scanner.

Stack obligatoire :
- Vite.js ;
- Tailwind CSS v4 ;
- JavaScript Vanilla.

Interdits :
- React ;
- Vue ;
- Angular ;
- jQuery ;
- Bootstrap ;
- Backend ;
- Base de données.

Architecture annoncée :

src/

Règle : toujours réutiliser ces données et ne jamais dupliquer les informations ailleurs.

Règles de codage :
